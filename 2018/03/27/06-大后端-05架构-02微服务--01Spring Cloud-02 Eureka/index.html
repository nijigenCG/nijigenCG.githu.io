服务注册与发现

服务注册：大量应用地址使用手工维护是不便的，通过注册中心，各服务将自己的地址注册到服务中心，产生一个服务清单，注册中心也会以心跳链接的方式检测服务可用性

服务名   位置
服务A    192.168.0.1：8080、192.168.0.12：8080
服务B    192.168.0.101：8080、192.168.0.102：8080

服务发现：服务c调用服务a，将向注册中心发送请求，c便可获取到a位置，以轮询策略取出一个位置进行服务调用。这就是服务端负债均衡。

Eureka服务治理使用RESTful API，也支持非java语言构建的微服务。Eureka可以单机模式部署，也可以集群部署。集群部署时，一台宕机，通过分片可以继续提供服务，不想Zookeeper会停下来进行选举。当主机恢复后，会将分片的数据同步进来。

注册中心高可用化

做的注册中心就一台，如果宕机，将无法提供服务注册发现。Eureka设计就考虑高可用问题，所有节点都是提供方也是消费方，服务中心也是！
单机模式我们禁止注册中心自己注册自己

eureka.client.register-with-eureka=false
eureka.client.fetch-registry=false

实际生产环境不会这样处理，而是形成注册中心相互注册。
现在模拟一下多注册中心。peer1和peer2
拿到之前那个注册中心添加2个配置文件

1）application-peer1.properties
spring.application.name=eureka-server
server.port=8765
eureka.instance.hostname=peer1
eureka.client.serviceUrl.defaultZone=http://peer2:8766/eureka

2）application-peer2.properties
spring.application.name=eureka-server
server.port=8766
eureka.instance.hostname=peer2
eureka.client.serviceUrl.defaultZone=http://peer1:8765/eureka

我们还需要改hosts文件将127.0.0.1指向peer1和peer2
127.0.0.1 peer1
127.0.0.1 peer2

然后我们打包成jar，把jar复制成两个，一个叫peer1.jar；一个叫peer2.jar
cmd进入目录

java -jar peer1.jar --spring.profile.active=peer1
java -jar peer2.jar --spring.profile.active=peer2

可以看到注册中心的分片！

Eureka架构及详细配置

provider

服务注册
服务提供者会启动时发送REST请求注册同时带上元数据信息（双层Map结构）
服务同步
服务注册者可能注册到不同的注册中心上，信息被两个服务注册中心维护。当服务注册到一个注册中心时，请求会被转发到其他注册中心，实现同步。
服务续约
注册完成后，服务提供者会维护一个心跳到注册中心，防止被剔除。可配置在提供者客户端

#服务器续约任务的调用间隔时间
eureka.instance.lease-renewal-interval-in-seconds=30
#服务失效的时间
eureka.instance.lease-expiration-duration-in-seconds=90

consumer

获取服务
启动消费者，回想注册中心发送REST，获取服务清单，EurekaServer会维护一份清单给客户端，默认30秒更新一次服务缓存

eureka.client.registry-fetch-interval-seconds=30

服务调用
客户端拿到服务列表，根据自己决定调用哪个服务，Ribbon默认采用轮询策略，实现客户端负债均衡
服务下线
服务关闭时，需要告诉consumer服务已经关闭，当服务实例正常关闭，会发一个下线请求REST到eureka

注册中心

失效剔除
服务实例不一定正常关闭比如宕机。这回eureka有一个定时任务，没60秒将当前清单超时没有续约的服务剔除
自我保护
本地调试会遇到eureka注册中心警告。实际上是触发了自我保护机制。将服务的注册信息保留！
这样就造成一个问题，消费者客户端会拿到不存在的服务器客户端实例！所以我们这里需要容错机制！比如（请求重试，断路由）

本地调试建议配置注册中心,关闭保护机制，即使剔除无用实例
eureka.server.enable-self-preservation=false

详细配置

Eureka基本上可以当一个现成的产品，基本不需要修改配置信息

1.服务注册相关配置

#注册中心地址,默认为http://localhost:8761/eureka/ ,可修改为我们想要的
eureka.client.serviceUrl=http://localhost:9999/eureka/
#高可用多个注册中心配置
eureka.client.serviceUrl=http://peer1:9999/eureka/,http://peer2:9859/eureka/
#安全令牌，注册时需要校验方可注册
eureka.client.serviceUrl=http://<username>:<password>@localhost:9999/eureka/
...

2.服务实例相关配置

服务实例名称，对于一台主机多个实例的情况，默认规则如下
${spring.cloud.client.hostname}:${spring.apllication.name}:${spring.apllication.instance_id}:${server.port}
